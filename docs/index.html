LeetCode Test Case Generator

Overview

This project demonstrates that testing a solution can be as challenging as building the solution itself. We use LeetCode problems as real examples to explore sophisticated test case generation algorithms that avoid "circular proof" while ensuring comprehensive coverage.

Test case generation is difficult because common approaches lead to circular reasoning. For example, comparing the output of one cycle detection implementation against another requires trusting that the reference implementation is correct‚Äîbut that's exactly what we're trying to verify.

This challenge is known as the Oracle Problem in software testing‚Äîa well-established research area spanning several decades. Academic research has developed approaches like metamorphic testing to establish correctness through mathematical properties rather than reference implementations.

Goals

This project is:
  • An educational demonstration: Real examples showing that test generation can be as complex as the original problem
  • A bridge between theory and practice: Concrete implementations of Oracle Problem solutions and metamorphic testing concepts
  • A case study collection: Detailed analysis of testing challenges for specific algorithmic problems
  • An algorithmic exercise: Treating test case generation as a first-class algorithmic challenge worthy of study

Plan

The plan is to cover the first 200-300 LeetCode problems, working on one problem at a time. Each problem implementation includes the test case generation algorithm, analysis of why naive approaches fail, and documentation of the underlying mathematical principles.
